## 一、部署流程概览

```
开发者本地 git push master
    ↓
GitHub Actions 触发 (.github/workflows/deploy.yml)
    ↓
Build Job: checkout → npm ci → vitepress build → upload artifact
    ↓
Deploy Job: 拉取 artifact → deploy-pages 部署到 GitHub Pages
    ↓
https://01-xzl.github.io/Kether/ 可访问
```

## 二、控制部署的代码文件

### 2.1 核心：GitHub Actions 工作流

**文件**：`.github/workflows/deploy.yml`

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 触发分支 | `master` | 仅 push 到 master 分支触发 |
| 手动触发 | `workflow_dispatch` | 可在 GitHub Actions 页面手动运行 |
| Node 版本 | `20` | CI 容器内 Node.js 版本 |
| 构建命令 | `npm run docs:build` | 调用 package.json 脚本 |
| 产物目录 | `docs/.vitepress/dist` | 构建输出，上传为 artifact |
| 并发策略 | `cancel-in-progress: false` | 不取消正在运行的 job，避免部署中断 |

**完整内容**：
```yaml
name: Deploy VitePress site to GitHub Pages

on:
  push:
    branches:
      - master          # ← 变更此项可切换触发分支
  workflow_dispatch:     # ← 允许手动触发

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0   # 拉取完整 Git 历史（VitePress 需要 lastUpdated）

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci        # 严格按 lockfile 安装，比 npm install 更快更稳定

      - name: Build with VitePress
        run: npm run docs:build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist  # ← 构建产物目录

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 2.2 构建脚本：package.json

**文件**：`package.json`

```json
{
  "scripts": {
    "docs:dev": "vitepress dev docs",       // 本地开发 (HMR)
    "docs:build": "vitepress build docs",    // 生产构建 → docs/.vitepress/dist/
    "docs:preview": "vitepress preview docs"  // 本地预览构建结果
  }
}
```

### 2.3 Base 路径：config.mts

**文件**：`docs/.vitepress/config.mts`

```ts
base: '/Kether/',   // ← 部署到子路径的 key 配置
```

| 场景 | base 值 | 部署 URL |
|------|---------|---------|
| 仓库 `01-xzl/Kether` | `/Kether/` | `https://01-xzl.github.io/Kether/` |
| 如改用根域名账号 `01-xzl.github.io` | `/` | `https://01-xzl.github.io/` |

> **注意**：修改 `base` 后，`NavBar.vue`、`PostList.vue` 中所有 `withBase()` 调用会自动适配，无需手动修改。

## 五、常见问题与解决

### Q1: push 后 Actions 没触发？
- 确认 push 到 `master` 分支（不是 `main`）
- 检查 `.github/workflows/deploy.yml` 文件存在

### Q2: 部署成功但白屏？
- `config.mts` 中 `base` 值必须与仓库名一致 → 当前 `'/Kether/'`
- 确认所有链接使用 `withBase()` 而非硬编码

### Q3: `npm ci` 失败？
- 确保 `package-lock.json` 已提交
- CI 环境需 Node 20+

### Q4: 部署 404 错误？
- GitHub Pages 未启用（需在 Settings 中开启）
- `deploy-pages` action 需要 `pages: write` + `id-token: write` 权限
